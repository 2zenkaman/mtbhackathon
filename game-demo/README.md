## MTB Bank Tycoon — demo backend (for Godot)

Что в демке:
- **Прогресс/апгрейды**: кассиры, банкоматы, расширение сейфа, ручное обслуживание.
- **Очередь + лимит хранилища (сейф)**: доход копится в сейф до лимита; переполнение мотивирует заходить.
- **Стрик (🔥) + ежедневный чек‑ин**: награда растёт со стриком, пропуск сбрасывает.
- **API под Godot‑клиент**: серверное состояние и расчёт офлайн‑прогресса.

### Запуск

```bash
cd mtbhackathon/game-demo
go run ./cmd/server
```

Проверка, что сервер жив:
- `GET http://localhost:8088/healthz`

### API (для Godot)
- `GET /api/state` — получить состояние игрока.
- `POST /api/action` — применить действие:
  - `buy_cashier`
  - `buy_atm`
  - `upgrade_vault`
  - `collect_vault`
  - `serve_manual`
- `POST /api/checkin` — ежедневный чек‑ин и награда за стрик.

### Примечания (для хакатона)
- Данные игрока в памяти сервера (для простоты). Идентификатор игрока передаётся заголовком `X-Player-Id`.
- Логику офлайн‑дохода/очереди считает сервер на каждом запросе (античит‑подход).
